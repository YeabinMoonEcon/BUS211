Insertion Sort

  • Let's analyze insertion sort
  • The running time depnds on the input
    • Two factors
      • input size
      • initial position
      • Assume input size as the number of items in the input
      • The input size $n$ will impact on the running time more and more
  • Array $A$ contains the values to be sorted
  • The number $n$ of values of sort
  • The values occupy positions $A[1]$ through $A[n]$
    • \[\begin{aligned} T(n) &= c_1n + c_2(n-1)+c_4(n-1) + c_5 \sum_{i=2}^{n}t_i \\ &+c_6 \sum_{i=2}^{n}(t_i-1)+c_7 \sum_{i=2}^{n}(t_i-1) \\ &+c_8(n-1) \end{aligned} \]

    • Consider the input list is sorted already
      • $A[i]$ is greater than $A[1:i-1]$
      • $t_i$ = 1
      \[\begin{aligned} T(n) &= c_1n+c_2(n-1)+c_4(n-1)\\ &+c_5(n-1)+c_8(n-1) \end{aligned} \]
    • This is the best case
      • The running time is a linear function of $n$
    • Think about the worst case
      • the reverse sorted order
      • For each $A[i]$, need to consider all $A[1:i-1]$
    • So $t_i=i$, for $i=2,3,...,n$
      • need to calculate $\sum_{i=2}^{n}i$ and $\sum_{i=2}^{n}(i-1)$
      • since the while loop exists only when $j$ reaches 0
      \[\begin{aligned} T(n) &= c_1n+c_2(n-1)+c_4(n-1)\\ &+c_5(\frac{n(n+1)}{2} -1)+c_6(\frac{n(n-1)}{2})\\ &+c_7(\frac{n(n-1)}{2})+c_8(n-1) \end{aligned} \]
    • This is the worst case
      • The running time is a quadratic function of $n$
      • $an^2+bn+c$
    • Usually concentrate on finding only the worst-case running time. Why?
      • It gives an upper bound on the running time for any input
      • The worst case occurs faily often
      • The average case is often roughly as bad as the worst case
    • In practice, ignore the actual cost of each step, $c_k$
      • express $an^2+bn+c$
      • $c_k$ would provide more detail than we need
      • consider only the leading term of a formula $an^2$
        • the lower-order tems are relatively insignificant for large $n$
    • For example, suppose an algorithm implemented takes $0.01 n^2 + 100n$
      • $0.01 n^2$ dominates the $100n$ term once $n$ exceeds 10,000
      • Many real-world problems have much larger input sizes

    Order of growth

    • This expression is called the order of growth
    • Use the Greek letter $\Theta$ (theta)
      • insertion sort has a worst-case running time of $\Theta(n^2)$
        • roughly proportional to $n^2$ when $n$ is large
      • insertion sort has a best-case running time of $\Theta(n)$
        • roughly proportional to $n$ when $n$ is large
    • Usually consider one algorithm to be more efficient than another if its worst-case running time has a lower order of growth
    • Due to constant factors and lower-order terms, hard to predict small input algorithm
    • There is always some number $n_0$ such that for all input size $n\ge n_0$, $\Theta(n^2)$ beats $\Theta(n^3)$ in the worst case
